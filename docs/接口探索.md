## 接口详情与验证结果

### 1. 🥇 Apple iTunes Search API — 首选：中文元数据+试听

**特点**: 无需注册，华语曲库最强，返回30秒试听音频

**验证结果**: ✅ 2026-07-12 实调确认
```
GET https://itunes.apple.com/search?term=稻香 周杰伦&media=music&country=cn&limit=1
→ 返回: trackName="稻香", artistName="周杰伦", collectionName="魔杰座"
  artworkUrl100 → 替换为600x600bb 得高清封面
  previewUrl → 30秒 AAC 试听片段 (154字符URL, 有效)
  trackTimeMillis=223453ms
```

**端点和字段**:

| 端点 | 说明 |
|:-----|:------|
| `GET /search?term={关键词}&media=music&country=cn` | 搜索歌曲 |
| `GET /lookup?id={collectionId}&entity=song&country=cn` | 专辑详情+曲目列表 |

| 关键返回字段 | 说明 |
|:-------------|:------|
| `trackName` | 歌曲名 |
| `artistName` | 歌手名 |
| `collectionName` | 专辑名 |
| `releaseDate` | 发行日期 |
| `artworkUrl100` | 封面 100x100（改后缀得大图） |
| `trackCount` | 专辑总曲目数 |
| `trackNumber` | 当前曲目编号 |
| `primaryGenreName` | 流派 |
| `previewUrl` | **30秒试听片段**（m4a格式） |
| `trackTimeMillis` | 歌曲时长（毫秒） |

**封面尺寸**: `artworkUrl100` 中的 `100x100bb` → `600x600bb` 或 `1200x1200bb`

---

### 2. 🥇 LRCLIB — 首选：同步歌词（LRC格式）

**特点**: 免费、无限制、无需注册，专注歌词，支持同步LRC歌词，中文曲库好

**验证结果**: ✅ 2026-07-12 实调确认
```
GET https://lrclib.net/api/search?track_name=稻香&artist_name=周杰伦
→ 返回20条结果，含2个版本的《魔杰座》同步歌词（摩杰座/魔杰座）
  每个结果包含 syncedLyrics 字段（[MM:SS.xx]格式）

GET https://lrclib.net/api/get?artist_name=周杰伦&track_name=稻香&album_name=魔杰座&duration=243
→ 返回54行同步歌词，从 [00:31.05] 到 [03:11.11]
  完美进入歌词，instrumental=false
```

**端点**:

| 端点 | 说明 |
|:-----|:------|
| `GET /api/search?track_name=&artist_name=` | 搜索歌词记录 |
| `GET /api/get?track_name=&artist_name=&album_name=&duration=` | 精确匹配歌词 |
| `GET /api/get-cached?track_name=&artist_name=&album_name=&duration=` | 仅查本地缓存 |
| `GET /api/get/{id}` | 按ID获取 |

**响应字段**: `id`, `trackName`, `artistName`, `albumName`, `duration`(秒), `instrumental`, `plainLyrics`, `syncedLyrics`

> **注意**: duration 参数很关键，LRCLIB 需要 ±2秒 的时长匹配才能返回歌词。建议先获取歌曲时长，再传近似值（容差自动处理）。

### 3. 🥈 MusicBrainz + Cover Art Archive — 补充：深层元数据+ID解析

**特点**: 元数据最丰富（ISRC、MBID、创作者），适合ID解析和深层元数据

**验证结果**: ✅ 2026-07-12 实调确认
```
GET https://musicbrainz.org/ws/2/recording?query=artist:周杰伦 AND recording:稻香&fmt=json
→ 返回54条结果，含完整artist-credit、MBID、时长
```

**端点**:

| 端点 | 说明 |
|:-----|:------|
| `GET /ws/2/recording?query={Lucene}&fmt=json` | 搜索录音（含专辑关联） |
| `GET /ws/2/release?query={Lucene}&fmt=json` | 搜索专辑 |
| `GET https://coverartarchive.org/release/{release_mbid}/front` | 封面（302跳转） |
| `GET https://coverartarchive.org/release/{release_mbid}/` | 封面列表JSON |

**头部要求**: `User-Agent: NasMusicTV/1.0 (your-email@example.com)`

### 4. 🥉 Meting-API — 中文深度补充（自建推荐）

**特点**: 对接网易云/QQ音乐，中文全覆盖，含歌词+封面+播放链接

**验证结果**:
- `mo-app.cn`: ⚠️ 触发 Google 418 (I'm a teapot) 反爬
- `musictsapi.chuye.us.kg`: ❌ DEPLOYMENT_NOT_FOUND (已下线)
- `meting.mikus.ink`: ❓ 待验证（mikus-loli/Meting-API 有 Docker 镜像）

> **强烈建议**: 自建 Meting-API (Docker)，代码: `xizeyoupan/Meting-API` 或 `mikus-loli/Meting-API`

**Docker 自建**:
```bash
docker pull ghcr.io/mikus-loli/meting-api:latest
docker run -d --name meting -p 3000:3000 ghcr.io/mikus-loli/meting-api:latest
```

**支持平台**: netease（网易云）, tencent（QQ音乐）, kugou（酷狗）, kuwo（酷我）, migu（咪咕）, bilibili

**API格式**: `GET /api?server={平台}&type={类型}&id={ID}`

| 类型 | 说明 |
|:-----|:------|
| `type=search&id=关键词` | 搜索 |
| `type=song&id=歌曲ID` | 歌曲信息 |
| `type=url&id=歌曲ID` | 播放链接 |
| `type=lrc&id=歌曲ID` | 歌词 |
| `type=pic&id=歌曲ID` | 封面 |

## 推荐集成方案 (v3.0)

### 核心架构

```
                     ┌─ iTunes Search API ──→ 主搜索：元数据+封面+试听
                     │    成功→返回结果
MetadataSearchService─┤    失败→降级
                     │
                     ├─ LRCLIB ───────────→ 获取同步歌词（无限制）
                     │
                     ├─ MusicBrainz ──────→ 补充搜索+ID解析
                     │     + Cover Art Archive → 封面
                     │
                     └─ Meting-API(自建) ─→ 中文深度补充（可选）
                                       → 获取播放链接（试听）
```

### 歌词获取流程

```
1. 用户播放歌曲 → 获取 song.title, song.artist, song.album, song.duration
2. 请求 LRCLIB: GET /api/search?track_name={title}&artist_name={artist}
   └─ 成功 → 取最佳匹配的 syncedLyrics
   └─ 失败 → 尝试 GET /api/get?track_name={title}&artist_name={artist}&album_name={album}&duration={sec}
   └─ 仍失败 → 无歌词，显示"暂无歌词"
3. 缓存歌词结果到本地（避免重复请求）
4. 用户拖动歌词 → 更新本地LRC时间戳 → 保存到歌词缓存
```

### 简化的包依赖方案（推荐）

不再自己造Provider轮子，直接引入 **musicmeta** Kotlin 库：
```kotlin
// build.gradle.kts
implementation("io.github.famesjranko:musicmeta-core:最新版")
```

或自己实现轻量级方案（推荐）：

| 数据需求 | 首选API | 备用API |
|:---------|:--------|:--------|
| 歌曲搜索+封面 | iTunes Search | MusicBrainz + CAA |
| 同步歌词 | LRCLIB | Meting-API |
| 试听音频 | iTunes Preview(30s) | Meting-API |
| 歌手信息 | MusicBrainz | Wikipedia |
