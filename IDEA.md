# 音频文件管理工具 — 需求摘要

> 从方案 v5.2 整理，项目目录：`E:\temp\NasAudio\AudioFileManager\`

## 一句话目标

制作一个 Windows 桌面软件，自动整理散乱的音乐文件、统一文件名、自动搜索补充缺失的元数据（专辑/年代/封面/歌词）。

## 功能需求

### 核心流程（自动批处理）

```
选择源文件夹 → 递归扫描所有子文件夹 → 文件名解析(歌手+歌曲名) → 
网络搜索补全元数据 → 写入ID3标签/封面/歌词 → 整理到目标目录
```

### 关键功能列表

| 功能 | 说明 |
|:-----|:------|
| **递归扫描** | `os.walk()` 遍历所有子文件夹，跳过隐藏目录；文件列表按文件夹分组展示 |
| **文件名解析** | 从各种文件名格式自动提取歌曲名和歌手（支持 `歌曲名-歌手` / `歌手 - 歌曲名` / `01_歌曲名-歌手` 等） |
| **元数据搜索** | 按可配置优先级链式搜索：iTunes → LRCLIB(歌词) → MusicBrainz → Meting-API(中文补充)；免费公开 API，无需申请 |
| **手动搜索** | 详情面板可手动点「🌐 网络搜索信息」，异步并行搜索多条结果，◀ ▶ 切换预览后填入 |
| **ID3 写入** | 歌曲名/歌手/专辑/年份/流派/曲目号写入标签，封面保存为 cover.jpg，LRC 歌词文件 |
| **目录整理** | 按 `歌手→专辑(年份)→歌曲` 自动归类，支持复制/移动两种方式 |
| **试听** | 弹窗播放本地文件，波形进度条可拖拽，滚动歌词，可保存调整后的歌词时间轴 |
| **元数据编辑** | 详情面板所有字段可编辑，修改后点保存写入文件标签 |

### 设置项

| 设置 | 选项 |
|:-----|:------|
| 输出根目录 | 用户自定义整理后文件的存放位置 |
| 搜索接口 | 拖拽排序（启用/禁用）、超时配置、Meting-API 地址自定义 |
| 整理后删除源文件 | 勾选后改为移动（否则为复制） |
| 元数据保存方式 | 仅标签 / 仅独立文件 / 两者都写 |
| 文字编码统一 | 可选启用，目标字符集 UTF-8/GBK/GB18030/ISO-8859-1 |
| 文件名冲突处理 | 保留第一个 / 保留最后一个 / 保留音质最好的(自动对比比特率/采样率/格式/声道) / 自动重命名加序号 / 每次询问 |
| 内容重复检测 | 跳过 / 覆盖 / 保留全部 / 询问；可选启用 SHA-256 哈希检测 |

### 工程稳健性

| 机制 | 说明 |
|:-----|:------|
| **断点续传** | `batch_state.json` 逐文件记录状态，崩溃/停止后下次可选择继续 |
| **停止按钮** | 当前文件允许完成，剩余的标记 pending，不丢失进度 |
| **文件名冲突** | 不同版本同一首歌整理后同名 → 5 种处理模式 + `AudioQuality` 智能评分选最优 |
| **内容重复** | SHA-256 哈希检测完全相同文件，支持 skip/overwrite/keep_all/ask |
| **网络请求速率控制** | 各 API 独立限速策略：iTunes 可少量并发、LRCLIB 可并发、MusicBrainz 严格 1请求/秒（官方要求）、Meting-API 严禁并发+等待500ms~1s；429 自动回退 60s+指数重试 |

### 网络请求速率控制策略

| Provider | 允许并发 | 建议频率 | 被限后恢复 |
|:---------|:--------:|:---------|:-----------|
| iTunes Search | ✅ 2 | 无延迟 | 自动重试 1 次 |
| LRCLIB | ✅ 3 | 无延迟 | 等待 60 秒后重试 |
| MusicBrainz + CAA | ❌ 1 | **每秒最多 1 次** | 等待 Retry-After |
| Meting-API (公开) | ❌ 1 | 每次后等待 500ms~1s | 等待 30~60s |
| Meting-API (自建) | ✅ 2 | 间隔 200ms | 自动重试 1 次 |

> MusicBrainz 官方硬性要求：https://musicbrainz.org/doc/XML_Web_Service/Rate_Limiting — 违反会导致 IP 被临时封禁。

## 技术选型

| 组件 | 选型 |
|:-----|:------|
| UI | PyQt6 |
| 音频播放 | PyQt6.QtMultimedia (QMediaPlayer) |
| 音频元数据 | mutagen |
| 网络请求 | httpx (异步) |
| 图片处理 | Pillow |
| 打包 | PyInstaller |
| 配置 | JSON 文件 (`%APPDATA%/AudioFileManager/`) |

### 窗口协作关系

| 窗口 | 从哪打开 | 与主界面的交互 |
|:-----|:---------|:---------------|
| **设置对话框** | 工具栏 `⚙️ 设置` 按钮 | 模态对话框，修改后立即写入 config.json；搜索接口配置即时生效，不需重启；目录整理/编码配置是下次批处理才读取 |
| **试听弹窗** | 文件列表双击或右键→试听 | 非模态（可同时操作主界面）；弹窗内修改的歌词点击 `💾 保存编辑` 后写入文件标签和同目录 LRC；试听窗口关闭后不影响主界面的元数据 |
| **搜索结果预览** | 详情面板 `🌐 网络搜索信息` 按钮 | 预览结果嵌入在主界面详情面板内（不单独弹窗）；点击 `✓ 填入元数据` 后数据填入详情面板各输入框，用户确认后点击 `💾 保存修改到文件` 才写入文件 |
